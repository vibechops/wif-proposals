# WIF program shapes

Reusable patterns from past proposals. Adapt modules per partner; do not copy client-specific details.

Reference implementation: `google/` (Google Flow x WIF India, modular 4-block program).

---

## Pattern A: Roadshow + finale

**Best for:** Product launches, tool education, brand activations at film schools.

| Block | Typical content |
|---|---|
| 1. Workshop events | City tour, expert-led demos, networking |
| 2. Open call | Scripts, pitches, or applications; WIF curates |
| 3. Production sprint | Filmathon / hackathon with tool access |
| 4. Awards / showcase | Mumbai finale, jury, Top 3, PR |

**Pricing:** Option A = events only. Option B = full funnel through awards.

---

## Pattern B: Single flagship event

**Best for:** Smaller budgets, pilot partnerships, one-city activations.

One combined event + content package + report. No national call unless budget allows.

---

## Pattern C: Content + community (no live events)

**Best for:** Remote-first partners, async programs.

Masterclasses, member-only access, curated showcase online. Lower production cost.

---

## Modular proposal rule

When a partner has a US/global approval chain:

- Pitch each module as a standalone block
- Let them approve 1+2 first, add 3+4 later
- Keep film/content rules in a single appendix slide

Google example: Modules 1-4 in `google/program-flow.md`.

---

## Standard WIF value props (reuse in decks)

- 3,500+ WIF India member outreach base
- Film institute relationships (WWI, FTII, SRFTII, etc.)
- Rabia-led program management and industry engagement
- Guneet Monga Kapoor / WIF LA credibility
- End-to-end production: venues, curation, finale events

---

## Typical open decisions (check every proposal)

- Option A vs B scope and fee
- Who funds participant tool access
- Number of cities and venues
- National vs city-limited submissions
- Production window (7 vs 14 days)
- Prize structure and jury composition
